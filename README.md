# 멀티캠퍼스 "건강기능식품 추천 AI 챗봇"

# 1. 프로젝트 개요
"헬스 메이트"는 AI 기반의 챗봇을 누구나 사용할 수 있도록 하는 챗봇 구현을 목표로 진행한 프로젝트입니다.

### 문제 인식 및 목표
현대 사람들은 "건강"이라는 키워드에 관심을 가지고 있습니다.
이에 따라 운동, 식단 관리 이외에도 각종 영양제와 같은 건강식품의 수요도 자연스럽게 상승할 것이라고 생각했습니다.
하지만 건강기능식품을 처음 접하는 사람들은 자신에게 맞는 제품인지 확인하지 못하고 구매를 하는 경우가 많습니다.

그래서 건강기능식품 구매하고자 하는 사람들에게 실제 리뷰를 바탕으로 자신에게 딱 맞는 제품을 추천하고자 했습니다.

# 3. 구현 과정
프로젝트는 데이터 수집 -> 전처리 -> 감정분석 및 리뷰 요약 -> RAG 구현 -> 웹 페이지 구현 으로 진행되었습니다.

### 3-1. 데이터 수집
- 각 플랫폼에서 카테고리별 제품 30개에 대한 리뷰, PDF문서, 영양정보 수집
- 리뷰를 제품별로 그룹화하고 구분자(/)를 사용하여 하나의 레코드로 병합
- PDF문서는 PDFLoader를 활용해 로드하고 TextSplitter를 사용하여 청크 분할

### 3-2. 전처리
- 정규표현식을 사용하여 특수문자, 반복되는 표현 등 제거
- 컬럼명 통일 후 모든 데이터 병합

### 3-3. 감정분석 및 리뷰 요약
- GCP gemini API를 활용하여 긍정, 부정, 중립 리뷰 분류
- 주요 키워드 추출 및 요약츨 통해 많이 등장한 건강 기능과 특징 정리
- 리뷰 임베딩 후 코사인 유사도 계산

### 3-4. RAG 구현
- LlamaIndex 기반 RAG 시스템 구현
- 코사인 유사도 기반 상위 후보 리뷰에 대한 제품 추천

### 3-5. 웹 페이지 구현
- Streamlit을 사용하여 RAG 연결 및 웹페이지 구현
- 평균 응답 속도 약 2초 이내

# 4. 향후 개선 사항

1. 출력 속도 향상
   현재 출력 속도는 평균 2초 정도로 확연히 느린 속도는 아니지만 데이터가 더 많아졌을 경우를 대비해서 벡터 데이터베이스를 활용해 출력 속도를 높이는 최근접 탐색 라이브러리를 도입할 계획입니다. 해당 기술을 적용하면 사용자가 더욱 빨리 출력을 받아볼 수 있을 것이라 기대됩니다.

2. 개인화 추천 고도화
   현재는 사용자의 질문에 가장 유사한 제품을 추천하고 있지만 더 나아가 GMM 기반 소프트 클러스터링 기법을 적용하여 여러 관심사 그룹에 동시에 속할 수 있도록 모델링할 계획입니다. 해당 기술을 적용하면 사용자의 복합적인 니즈를 파악해 더 정교한 맞춤형 제품을 추천할 수 있을 것입니다.

3. 카테고리 및 제품 추가
   현재의 모델은 MVP모델로 카테고리와 제품의 수가 비교적 적습니다. 차후 더 많은 카테고리와 그에 맞는 제품의 데이터를 수집하여 더 많은 선택지를 주고자 합니다.

# 5. 후기
이번 프로젝트를 통해  단순한 모델 학습만이 아니라 프로젝트 기획부터 웹 페이지 구현까지 엔드 투 엔드 파이프라인을 설계하고 구현할 수 있었습니다.
이러한 과정에서 모델 개발뿐만아니라 데이터 처리 방식, 모델 선택, 검색 구조가 모두 연결되어야 의미 있는 결과가 나온다는 것을 배웠습니다. 이를 통해 단순히 AI 모델 개발이 아니라 하나의 시스템을 완성하는 시각을 가지게 되었습니다. 

또한 프로젝트를 관리하는 팀장으로써 가장 어려웠던 점은 팀원들의 작업 속도와 생각이 다르다는 것이였습니다.
처음에는 역할을 나누고 태스크를 나눠서 기간을 정하는 것만으로도 충분할거라고 생각했습니다. 하지만 프로젝트를 진행하는 도중 각 태스크가 유기적으로 연결되지 않아 여러 차례 충돌이 있었습니다. 이를 해결하기 위해 마무리 하기 전 간단한 각자 한 일에 대해 공유하고 주 단위 체크리스트를 작성하여 일정을 관리했습니다. 또한 의견 충돌로 문제가 생기면 즉시 협의할 수 있도록 하였습니다. 그 결과 팀장은 지시하는 사람이 아니라 문제 해결을 돕는 조율자라는 점을 배울 수 있었습니다.
